# Retrieval eval harness

> Трек A, шаг 1. «Настраивать на глаз нельзя — нужно мерить.» Этот харнес даёт
> числа (**Recall@k / MRR / nDCG@k**) на golden-наборе из реальных документов,
> чтобы любое изменение ретрива (чанкинг, fusion, rerank, HyDE, late-interaction)
> сравнивалось метрикой, а не ощущением. Разблокирует трек C.

## Файлы

| Файл | Назначение |
|---|---|
| `goldset.toml` | Ground truth: пары «вопрос → правильный чанк». **Единственное, что правишь руками.** |
| `resolve.py` | Резолвит якоря набора → актуальные `chunk_id` из Qdrant. Плюс режим валидации. |
| `retrievers.py` | 4 стратегии: `dense` / `sparse` / `hybrid` / `hybrid+rerank` → ранжированные `chunk_id`. |
| `metrics.py` | Чистые функции: `hit@k`, `recall@k`, `mrr`, `ndcg@k`. Покрыты `tests/test_eval_metrics.py`. |
| `run_eval.py` | Прогон + таблица + сохранение снапшота. |
| `results/*.json` | Снапшоты метрик (для сравнения «до/после»). |

## Запуск

```bash
# 1. Валидация набора — каждый якорь должен резолвиться в нужный чанк:
docker compose exec -T backend sh -c 'cd /app && PYTHONPATH=/app uv run python -m app.eval.resolve'

# 2. Прогон eval (сохранит снапшот в results/baseline.json):
docker compose exec -T backend sh -c 'cd /app && PYTHONPATH=/app uv run python -m app.eval.run_eval --save baseline'

# Флаги: --k 1,3,5,10   --only hybrid,sparse   --verbose   --save <name>
```

## Ключевое решение: якоря, а не UUID

`chunk_id` — это UUID, который **пересоздаётся при каждом re-ingest**. А трек C —
это сплошные переиндексации (новый чанкинг, parent-document, ColBERT). Поэтому
golden-набор указывает на чанк **стабильным якорем**: `file` + список `gold`
(уникальные кусочки текста). `resolve.py` находит, в каких чанках Qdrant сейчас
лежит этот текст. Набор привязан к **смыслу**, а не к технической случайности —
переживает любую переиндексацию.

### Формат записи `goldset.toml`

```toml
[[q]]
id = "n-plus-1"                       # уникальный slug
file = "interviev_short.md"           # в каком документе искать якорь
style = "keyword"                     # keyword | semantic | mixed (только для анализа)
question = "Що таке проблема N+1 запитів в ORM?"
gold = ["N+1 запитів"]                # каждый сниппет → чанк(и), где он есть (норм.: lower + collapse ws)
note = "1 запит на список + N запитов на пов'язані дані."
heading = "..."                       # опционально: подстрока heading-path, для дизамбигуации
```

Сниппет матчится регистронезависимо и со свёрнутыми пробелами (устойчиво к шуму
markdown). Один вопрос может иметь несколько релевантных чанков (несколько
сниппетов или один сниппет на стыке чанков из-за overlap) — `nDCG`/`recall` это
учитывают честно.

## Метрики — что именно меряем

- **Hit@k** — попал ли *хоть один* правильный чанк в top-k. Самое интуитивное: «нашли вообще?».
- **Recall@k** — какая *доля* правильных чанков в top-k (при одном эталоне = Hit@k).
- **MRR** — `1/ранг` первого правильного. Награда за то, что ответ на позиции 1, а не 5.
- **nDCG@k** — кредит *каждому* правильному чанку по его позиции, нормированный на идеальный порядок (0..1).

Каждая стратегия запрашивается **один раз** на вопрос на максимальную глубину;
все @k считаются срезом одного ранжированного списка → стратегии сравниваются на
идентичных выборках.

## Стратегии

- `dense` / `sparse` — диагностические бэйзлайны (одиночный вектор / только BM25). В проде не используются, нужны чтобы видеть вклад каждой стороны.
- `hybrid` — реальный прод-путь, RRF без rerank (`hybrid_search(rerank=False)`).
- `hybrid+rerank` — реальный прод-путь с cross-encoder (`hybrid_search(rerank=True)`).

## Как пользоваться при экспериментах трека C

1. `--save baseline` — снять текущие числа.
2. Внести **одно** изменение (например, `chunk_size`, новый чанкер, HyDE-узел) + re-ingest.
3. Перепрогнать `--save <experiment>`.
4. Сравнить `results/baseline.json` ↔ `results/<experiment>.json`. Фиксировать Δ в `LEARNING_NOTES.md`.

> Якоря резолвятся заново на каждый прогон, поэтому после re-ingest набор править
> не нужно — если только переформулировка чанков не «выбила» сниппет (тогда
> `resolve.py` это покажет ⚠, и сниппет надо обновить).
